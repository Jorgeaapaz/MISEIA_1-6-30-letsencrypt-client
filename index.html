letsencrypt-client

A Rust CLI implementing the ACME protocol (RFC 8555) to automate TLS certificate issuance from Let's Encrypt or a local Pebble test server.

GitHub GitLab

✨ Key Features

🔑 ECDSA P-256

Account & certificate keys generated with ES256 using the ring crate.

🌐 HTTP-01 Challenge

Spins up a temporary HTTP server on port 80 to serve the ACME token automatically.

📜 Multi-domain SAN

Single order covers multiple domains via Subject Alternative Names in the CSR.

🧪 Pebble Test Environment

Full Docker Compose setup with Pebble & challtestsrv for local end-to-end tests.

💾 PEM Storage

Saves privkey.pem, cert.pem, and fullchain.pem per domain under certs/.

🔁 Renew & Show

CLI subcommands to renew existing certificates or inspect stored cert details.

⚙️ ACME Flow (RFC 8555)

  1. Fetch Directory — GET the ACME directory to discover endpoint URLs.
  2. Get Nonce — HEAD newNonce to obtain a fresh replay-nonce.
  3. Create / Load Account — POST newAccount with JWK and ToS accepted.
  4. Create Order — POST newOrder with the list of domain identifiers.
  5. Fetch Authorizations — GET each authorization URL from the order.
  6. Solve HTTP-01 Challenge — Serve the token at /.well-known/acme-challenge/<token>.
  7. Notify Challenge Ready — POST to the challenge URL to trigger validation.
  8. Poll Order Status — Poll until the order reaches ready or valid.
  9. Generate CSR — Build a CSR with all domains in SAN using rcgen.
  10. Finalize Order — POST finalize with the DER-encoded CSR (base64url).
  11. Download Certificate — GET the certificate URL when order state is valid.
  12. Persist — Write PEM files to certs/<domain>/.

🚀 Quick Start

# Issue a certificate for one or more domains
acme-client issue \
  --domain test1.example.com \
  --domain www.test1.example.com \
  --acme-url https://acme-v02.api.letsencrypt.org/directory \
  --email admin@example.com \
  --output ./certs

# Renew an existing certificate
acme-client renew --domain test1.example.com --output ./certs

# Inspect a stored certificate
acme-client show --domain test1.example.com

# Run against local Pebble (test environment)
ACME_URL=https://localhost:14000/dir \
ACME_INSECURE_TLS=true \
acme-client issue --domain test1.example.com --email dev@local.test
